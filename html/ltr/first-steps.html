Passo 1: Cadastrar Usuário

Para que um novo projeto seja criado, é necessário o cadastro prévio do usuário que será o Administrador do Projeto.

Gerencie o cadastro de usuários no Menu X Submenu Y ou Clique aqui para ser direcionado à respectiva página.
Passo 2: Criar o projeto

O projeto é um escopo dentro do qual serão criadas as demais configurações.

Uma empresa pode ter um ou mais projetos, por exemplo:

A empresa Alfa Software atua na área de segurança e possui dois projetos:
1) Porteiro eletrônico e
2) Monitoramento veicular.

Crie ou edite um projeto no Menu X Submenu Y ou Clique aqui para ser direcionado à respectiva página.
Passo 3: Criar os produtos de determinado projeto

Produto é o componente que o cliente utiliza.

O chamado é aberto para determinado produto de algum projeto específico.

Cada produto possui tipos de solicitações que podem ser características apenas dele.

Crie ou edite os produtos do projeto no Menu X Submenu Y ou Clique aqui para ser direcionado à respectiva página.
Passo 4: Cadastrar os Itens de Configuração

Item de configuração é qualquer componente que necessite ser configurado com o objetivo de se entregar um serviço de TI.

Um Produto possui um ou mais itens de Configuração, normalmente o Item de Configuração que o cliente manipula ou está vendo possui o mesmo nome do produto.

Por exemplo:

  • O Sistema de Monitoramento Veicular possui os seguintes itens de configuração:
  • O Rastreador Veicular
  • O Sistema Configurador do Rastreador
  • O Gateway, responsável por decodificar as informações que vem do rastreador e enviá-las ao Sistema de Monitoramento.
  • O Sistema de Monitoramento Veicular, manipulado pelos usuários, que apresenta em tela as informações dos rastreadores ou veículos em tempo real.
  • O servidor de banco de dados X.
  • O servidor de banco de dados Y.
  • O servidor de aplicação.
    • e etc..

Normalmente o Item de Configuração é identificado durante o atendimento, o cliente não costuma saber qual item de configuração foi afetado, mas os atendentes treinados conseguem chegar a uma conclusão de acordo com a análise realizada.

Passo 5: Cadastrar os Tipos de Solicitações

O tipo de solicitação será o motivo pelo qual o usuário irá criar o chamado para determinado produto

Exemplos de tipos de solicitações para um sistema de Monitoramento Veicular

As Solicitações estarão separadas em dois níveis, assim como na lista abaixo.

Passo 6: Associoar Tipos de Solicitações aos Produtos

Cada produto contém sua particularidade e por isto pode necessitar de diferentes Tipos de Solicitações.

Passo 7: Criar os Status e Fluxos das solicitações

Fluxo é a definição dos estados possíveis que determinada solicitação pode ter e a ordem que este estados podem seguir.

Cada Tipo de Solicitação de determinado Projeto pode possuir um Fluxo específico.

O tipo de solicitação “Reporte de incidente: Falha” pode possuir diferentes fluxos dependendo do projeto, por isto a configuração é aplicável aos projetos conforme configuração.

Se o fluxo for criado para determinado projeto, ele estará visível para os demais projetos fazerem uma cópia, pois caso seja necessária alguma alteração do fluxo em alguns destes projetos, o outro não será afetado.

Passo 8: Criar a origem dos chamados

Os chamados possuem origens diversas.

Por padrão, esta ferramente possui 3 origens que não podem ser alteradas, são elas:

Chat: Quando o chamado foi aberto a partir de um chat que o cliente iniciou com algum atendente.
Portal do cliente: Quando foi aberto através do portal do cliente.
Telefone: Quando foi aberto devido a atendimento telefônico

A ferramenta permite que sejam criadas outras origens de chamados e inativadas as que já existem.
